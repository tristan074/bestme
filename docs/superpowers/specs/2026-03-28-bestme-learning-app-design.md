# BestMe - Fred 学习助手 Web 应用设计文档

## 概述

为小学二年级学生 Fred 构建的学习 Web 应用，涵盖数学和语文两大模块。Fred 在 iPad 上通过局域网独立使用，应用部署在 Mac Mini 上。

## 技术栈

- **框架**：Next.js (App Router)
- **样式**：Tailwind CSS（儿童友好的大按钮、大字体、鲜明配色）
- **数据库**：SQLite（通过 Prisma ORM）
- **语音**：Web Speech API（浏览器原生 TTS，用于语文听写播报）
- **部署**：Mac Mini 本地运行，iPad 通过局域网访问
- **后期路径**：React Native / Capacitor 转 App

## 页面结构

```
首页（选择模块）
├── 数学
│   ├── 专项选择（小九九 / 进位加减法 / 百以内加减法）
│   ├── 练习配置（题量选择，默认20题）
│   ├── 练习页面（计时 + 逐题作答）
│   └── 练习结果页
├── 语文
│   ├── 生字簿管理（新建/重命名/切换/归档）
│   ├── 生字库（按课文分组，管理学习状态）
│   ├── 今日任务（新学 + 到期复习）
│   ├── 听写页面（语音播报 → 纸上写 → 对答案 → 标记对错）
│   └── 生字批量导入
├── 我的数据
│   ├── 数学趋势（各专项正确率/用时曲线）
│   ├── 语文进度（未学/学习中/已掌握）
│   ├── 易错汇总（数学错题本 + 语文易错字）
│   ├── 积分 & 成就墙
│   └── 打卡日历
```

## 数学模块

### 三个专项

| 专项 | 题目范围 | 示例 |
|------|---------|------|
| 小九九 | 1×1 到 9×9 | `7 × 8 = ___` |
| 10以内进位加减法 | 结果≤20的进位加法及对应减法 | `6 + 7 = ___`、`13 - 5 = ___` |
| 百以内两位数加减法 | 两位数±一位数/两位数，结果≤100 | `47 + 36 = ___`、`82 - 25 = ___` |

### 练习流程

1. 选择专项 → 配置题量（可选10/20/30/50，默认20） → 开始
2. 屏幕显示大字号算式 + 数字输入框，顶部显示计时器
3. Fred 输入答案 → 立即判对错 → 自动跳下一题
4. 做完所有题 → 结果页：总用时、正确率、错题列表（显示正确答案）

### 出题规则

- 每次随机生成，同一次练习内不重复
- 错题加入错题本，后续练习中按比例混入（如20题中抽3-5题来自错题本）

### 数据记录

每次练习记录：
- 日期、专项类型、题量
- 总用时、每题用时
- 每题的算式、Fred的答案、是否正确

## 语文模块

### 生字数据结构

每个字包含：
- 汉字
- 拼音（自动生成）
- 所属课文/分组
- 学习状态：未学 → 学习中（进入复习周期）→ 已掌握

### 生字簿管理

- 支持创建多个生字簿（如"二年级下册"、"三年级上册"）
- 生字簿可重命名、切换、归档
- 每个生字簿独立管理生字和复习进度
- 当前激活的生字簿作为听写任务来源

### 批量导入

- 提供文本框，直接粘贴汉字（支持空格分隔或连续输入）
- 系统自动识别、去重、自动匹配拼音
- 可标注所属课文（如"第3课"），方便分组管理

### 生字库管理页面

- 按课文分组展示所有生字
- 清晰区分"未学"和"已学"状态
- 支持手动勾选/批量切换状态：未学 → 已学
- 标记为"已学"的字立即进入间隔复习周期

### 听写流程

1. 系统根据复习计划生成今日任务列表（到期复习的字）
2. 进入听写 → 语音播报第一个字的拼音/词语
3. Fred 在纸上手写 → 点"下一个" → 播报下一个字
4. 全部播完 → 系统显示所有正确答案（大字号清晰展示）
5. Fred 对照纸上结果，逐个标记 ✓ 对了 / ✗ 错了
6. 提交 → 系统记录结果，更新复习计划

### 间隔复习策略（基于艾宾浩斯遗忘曲线）

- 基础复习间隔：学习当天 → +1天 → +2天 → +4天 → +7天 → +15天
- **答对**：正常推进到下一个复习节点
- **答错**：重置间隔，从第1天重新开始
- 连续通过所有复习节点 → 标记为"已掌握"
- 每日任务优先级：复习 > 新学（任务量过大时，剩余顺延到明天）

## 积分与成就系统

### 积分规则

| 行为 | 积分 |
|------|------|
| 完成一次数学练习 | 10分 |
| 数学正确率100% | +5分 |
| 数学刷新个人最快用时 | +5分 |
| 完成一次语文听写 | 10分 |
| 语文听写全对 | +5分 |
| 每日打卡（数学+语文各至少一次） | 5分 |
| 连续打卡加成 | 3天+5、7天+10、30天+30 |

具体数值可后续调整。

### 成就徽章（举例）

- **闪电口算**：小九九20题1分钟内完成（侧重速度）
- **速算小能手**：任意专项刷新用时记录
- **百字英雄**：累计掌握100个生字
- **坚持之星**：连续打卡7天 / 30天
- **零失误**：单次练习全对

### 展示方式

- 首页显示当前积分总数 + 最近获得的成就
- "我的数据"里有完整的积分历史和成就墙

## 数据看板

### 数学板块

- 各专项近期趋势图（用时曲线 + 正确率曲线）
- 各专项个人最佳记录
- 错题本（按专项分类，可点击重练）

### 语文板块

- 生字总览：进度条（未学 / 学习中 / 已掌握）
- 今日复习任务完成情况
- 易错字列表（错过2次以上的字）

### 总览

- 打卡日历（热力图，一眼看连续性）
- 积分总数 + 积分趋势
- 成就墙（已解锁的徽章）
- 连续打卡天数

## 二期规划（不在一期范围内）

- 笔顺动画展示（对答案时展示每个字的笔顺）
- App 转换（React Native / Capacitor）

## 设计原则

- **儿童友好**：大字体、大按钮、鲜明色彩、简洁交互
- **iPad 优先**：触摸操作友好，适配平板屏幕
- **数据驱动**：所有练习行为都记录，用于趋势分析和复习策略
- **YAGNI**：一期只做核心功能，不过度设计
